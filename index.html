Nos dias atuais devido ao fato do uso de computadores estar tão difundido mundialmente fica difícil imaginar que ainda existem lugares que não tiram proveito das vantagens que a tecnologia oferece. Mesmo quando se considera somente o Brasil como um todo já é bastante comum o uso de computadores/ smartphones mesmo entre as classes mais carentes. Assim, foca difícil imaginar que ainda existem lugares que fazem muito pouco ou nenhum uso de softweres ou tecnologias mais modernas para gerenciar seus afazeres.


Um desses lugares, como descobri recentemente, é uma igreja local que apesar de ter um computador (e somente 1) a disposição, faz pouco uso das vantagens que ele pode oferecer, utilizandô-o somente para impressão e elaboração de algumas planilhas. Várias tarefas que poderiam se beneficiar de um controle e automação por software como agendamento de missas, casamentos, batisados, etc ou controle dos gastos/ dinheiro arrecadado pela igreja ou ainda os dados dos dizimístas, ainda são controlados por meio de anotações em pedaços de papel, agendas físicas e formulários impressos.


Na intenção de tentar mudar isso pelo menos um pouco e modernizar como esses processos são feitos e gerenciados surgiu a ideia de fazer um sistema gerencial pra automatizar ou pelo menos digitalizar parte deles. Assim comecei a desenvolver um projeto em python para buscar atenter ao máximo as necessidades que podem ser supridas por um sistema desse tipo. A linguagem Python foi escolhida por ser multiplataforma, visto que o computador da igreja esta rodando Windos 10 e o computador que uso para desenvolver está rodando Linux, e porque dentro das bibliotecas padrões da linguagem já tem uma biblioteca para desenvolvimento de interface gráfica chamada Tkinter.


O projeto ainda está nos seus estágios iniciais de desenvolvimento com apenas por volta de 5 dias a 1 semana de trabalho no código. Felizmente as coisas estão bem encaminhadas neste começo e eu considero finalizado a versão 1.0 da janela principal mesmo esta sendo a 1ª vez que trabalho com o Tkinter ou desenvolvo uma interface gráfica em geral. O sitema foi estruturado da seguinte forma: a janela principal é a janela que é aberta quando o programa é executado e ela é a resposável por gerenciar as janelas filhas que estão abertas, além disso, ela também é responsável por instalar e desinstalar módulos; módulo é o nome das janelas filhas e são elas que vão executar as funções que precisam ser executadas, a ideia é que cada módulo atenda a uma necessidade específica como por exemplo cadastro e gerênciamento de dizimístas.


Eu também coloquei o projeto em um repositório no Github para controle de versão e para torná-lo público, permitinndo a colaboração de outras pessoas interessadas em ajudar ou em utilizá-lo.


Como é possível perceber pelas capturas de tela é um programa bem simples e com funcionalidades básicas, porém dado o pouco tempo de desenvolvimento e o fato de que estou aprendendo enquanto desenvolvo, estou satisfeito com o resultado por enquanto. Os módulos começarão a ser desenvolvidos assim que obtiver mais especificações das tarefas que precisam ser executadas. Uma última consideração é a parte visual do projeto, esta ainda não está recebendo nenhum tipo de polimento, o máximo que esta sendo feito é a escolha da disposição dos elementos na janela, visto que esta não é a minha área de expertise ela ficará em segundo plano por um tempo, mas qualquer colaborção ou ajuda neste sentido é sempre bem-vinda.
